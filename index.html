Autonomous Data Ferrying from a Self-Organizing / Healing WSN in Disconnected Zones

SKY-MULE is a UAV-assisted autonomous data collection system for remote areas without fixed communication infrastructure. It uses low-power, solar-supported sensor nodes that self-organize into clusters and select the best Cluster Head based on energy, uptime, trust, and link quality. A UAV autonomously visits the area, wakes Cluster Heads, and collects stored environmental data wirelessly. Prototype testing confirmed successful Cluster Head election, scheduled communication, and end-to-end data delivery, achieving about 65% reliability in the initial firmware. Overall, SKY-MULE offers a scalable, low-cost, and energy-aware solution for long-term monitoring in disconnected environments.

Domain

Research domain and objectives

Literature Survey

WSNs support monitoring in remote areas but struggle without fixed infrastructure because gateways and multi-hop routing drain batteries. Data MULEs and UAVs improve connectivity by collecting buffered data, while LEACH/HEED clustering saves energy but lacks trust and security. Existing trust-aware methods are too heavy for low-cost nodes. BLE 5.0, ESP-NOW, deep sleep, solar power, and compression can improve range, communication, reliability, and lifetime.

Research Problem

Design a fully autonomous, energy-efficient, and secure WSN that can self-organize, aggregate data, and transfer it through UAVs without fixed infrastructure or human intervention.

Main Objective

Design and implement SKY-MULE, a self-organizing UAV-assisted WSN data ferrying system with long node lifetime and reliable end-to-end data delivery.

Key Objectives

  • Build solar-powered ESP32-S3 sensor nodes with power management and compression.
  • Develop secure CH election using battery, uptime, trust, and link quality.
  • Implement TDMA-based ESP-NOW communication and persistent storage.
  • Deploy UAV-based RF wake-up, Wi-Fi data collection, and dashboard visualization.

Methodology

The system has three layers:

  • WSN Layer: Sensor nodes collect data, elect secure CHs, and send data using ESP-NOW.
  • UAV Layer: UAV follows waypoints, wakes CHs using 433 MHz RF, and collects stored data via Wi-Fi/HTTPS.
  • Base Station Layer: Data is stored in PostgreSQL and visualized using a React dashboard.

Testing includes unit tests, integration tests, SITL simulation, and real field trials.
